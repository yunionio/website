# 推理模型库

管理模型来源、版本与缓存，供 Ollama/vLLM 等推理应用使用。

## 控制台中的推理模型库

入口 **人工智能 → 推理 → 推理模型库**。支持「安装模型数据」「导入社区模型」。创建 Ollama 等推理模板时需选择挂载的模型，实例运行后使用该模型。

## 模型库解决的问题

- **统一来源**：集中管理，多环境复用。
- **统一版本**：按版本管理，减少“同名不同版”差异。
- **复用与缓存**：减少重复下载与存储，提升部署效率。
- **权限与合规**：统一管理可见范围与使用边界。

## 模型生命周期

1. **导入/注册** — 登记名称、版本、来源。
2. **分发/缓存** — 节点侧按需缓存。
3. **使用** — 推理应用引用并加载。
4. **更新/回滚** — 版本管理。
5. **清理** — 回收历史版本与缓存。

## 与 Ollama / vLLM

- **Ollama**：本地缓存加载，注意磁盘与清理策略。
- **vLLM**：关注版本与发布（灰度/回滚）。

## 运维要点

- **缓存**：定期检查占用，避免磁盘占满。
- **版本**：建立命名与发布规范，明确默认版本与回滚。
- **离线**：内网/离线需提前准备来源与分发。

## 常见问题

### 模型拉取慢或失败

检查网络、带宽与权限；内网可提前离线导入。

### 磁盘占用持续增长

清理历史版本与缓存，为缓存目录预留空间并建立清理策略。

### 版本不一致导致效果差异

通过模型库统一版本，推理侧明确引用版本。
