# vLLM

vLLM 是高吞吐、低延迟的 LLM 推理服务，**需要 GPU**，适用于对并发与吞吐有要求的在线推理场景。

## 快速开始 {#quickstart}

创建 vLLM 推理实例大致分为以下几步：

1. **准备 GPU 环境**：完成 [配置 NVIDIA 与 CUDA 环境](../../getting-started/setup-nvidia-cuda)，并按 [AI模板](../llm-sku) 选择满足模型加载所需显存的 GPU 规格。
2. **创建推理实例**：控制台 **人工智能 → 推理 → 推理实例**，新建，选择 **vLLM** 推理模板创建实例。\n   - 平台通常会预置模板；若需自定义镜像/规格/模型等，再到 **人工智能 → 推理 → 推理模板** 新建或编辑。
3. **选择规格与镜像**：按需选择 [AI模板](../llm-sku) 与 [AI镜像](../llm-image)，重点关注 GPU 显存、CPU/内存与并发规模的匹配。
4. **选择模型与版本（如支持）**：若控制台支持关联 [AI模型库](../llm-instantmodel)，优先使用模型库统一管理模型来源与版本，并制定灰度/回滚策略。
5. **获取服务地址并验证**：进入实例详情页「连接信息」获取推理服务地址；结合运行状态与日志确认模型已成功加载且接口可用（具体调用方式以连接信息与镜像说明为准）。

## 配置

### 镜像与规格

- **规格选择**：参考 [AI模板](../llm-sku)，重点关注 GPU 显存、CPU/内存与并发规模之间的匹配。
- **镜像选择**：参考 [AI镜像](../llm-image)，使用明确版本（tag/digest）便于升级与回滚，并注意 CUDA/驱动兼容性。

### 模型与版本

- **模型库（如支持）**：结合 [AI模型库](../llm-instantmodel) 统一管理模型来源与版本；建议建立发布策略（灰度/回滚），避免生产流量直接命中未验证的新模型版本。
- **存储**：用于模型缓存、日志与（可选）权重文件落盘；建议使用持久化存储以提升重启后的恢复速度。

### 并发、吞吐与延迟

- **容量规划**：吞吐与延迟通常受 GPU 显存、并发与批处理策略影响；容量不足时可横向扩展实例或升级 GPU 规格。
- **网络**：推理接口对网络延迟敏感；建议业务侧与推理服务之间保持稳定低延迟链路，避免跨地域/高抖动链路。

### 变更与观测

- **升级与回滚**：通过 [AI镜像](../llm-image) 控制运行时版本；变更前评估 CUDA/驱动兼容性与模型版本兼容性。
- **观测**：关注 GPU 利用率、显存占用、QPS、P95/P99 延迟、错误率；结合日志定位模型加载失败、显存不足、请求超时等。

## 常见问题

### 启动后无法加载模型

- 检查模型来源与权限、网络连通性，以及模型文件是否完整可用。
- 检查规格是否满足模型加载所需的显存与内存峰值。

### 吞吐低或延迟高

- 关注 GPU 利用率与显存占用是否偏低/偏高，检查是否存在 CPU/网络瓶颈。
- 调整规格与并发策略，必要时横向扩容多个实例分摊流量。

### GPU 不可用或驱动不匹配

- 按 [配置 NVIDIA 与 CUDA 环境](../../getting-started/setup-nvidia-cuda) 检查驱动/CUDA 版本与节点状态。
