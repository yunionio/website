# Dify

Dify 是 LLM 应用开发与编排平台，支持构建对话、Agent 与工作流等应用。该应用本身不依赖 GPU，可在未配置 NVIDIA 环境的节点上运行。

## 快速开始 {#quickstart}

创建 Dify 应用实例大致分为以下几步：

1. **创建应用实例**：控制台 **人工智能 → 应用 → 应用实例**，新建，选择 **Dify** 模板创建实例。\n   - 平台通常会预置模板；若需自定义镜像/规格/参数，再到 **人工智能 → 应用 → 应用模板** 新建或编辑。
2. **配置推理与密钥**：在创建实例时配置下游推理（外部模型服务或 Ollama/vLLM），并配置相应凭据（建议使用容器密钥管理 API Key）。
3. **访问与初始化**：进入实例详情页「连接信息」获取访问地址，用浏览器打开完成初始化。
4. **验证**：在 Dify 中创建一个最小对话/工作流，发起一次调用，确认推理链路正常。

## 配置

### 镜像与规格

- **规格选择**：参考 [AI模板](../llm-sku)，主要关注 CPU/内存与并发规模。
- **镜像选择**：参考 [AI镜像](../llm-image)，使用明确版本（tag/digest）便于升级与回滚。

### 推理对接与密钥

- **外部推理**：可对接 OpenAI/月之暗面等外部模型服务；按控制台表单提示填写地址与鉴权信息。
- **对接推理实例（可选）**：如对接 Ollama/vLLM，确保推理实例已可用，并在 Dify 中配置其服务地址。\n  - 若下游为 GPU 推理服务，推理服务所在节点需按 [配置 NVIDIA 与 CUDA 环境](../../getting-started/setup-nvidia-cuda) 完成 GPU 环境。
- **容器密钥**：建议使用容器密钥管理 API Key，便于多实例复用与凭据轮换。

### 存储与数据

- **持久化**：建议使用持久化存储保存应用配置、知识库（如有）、日志与业务数据。
- **数据治理**：对知识库/数据集类数据建议建立备份与清理策略，避免数据长期膨胀影响性能。

### 网络与容量

- **网络**：确保 Dify 与其依赖组件（推理服务、数据库/向量库等）之间网络连通。
- **扩缩**：负载更多受 CPU/内存与外部依赖影响；扩容时建议同时评估下游推理服务容量。
- **升级与回滚**：通过 [AI镜像](../llm-image) 管理运行时版本；升级前建议备份关键数据与配置。
- **观测**：关注请求量、错误率、下游推理调用延迟、数据库/存储指标；结合日志定位依赖不可用、超时等问题。

## 常见问题

### 调用推理服务超时或失败

- 检查网络连通性与服务地址配置。
- 若下游为 GPU 推理服务，确认 GPU 节点环境正常（`nvidia-smi`）并检查推理服务容量。

### 数据增长导致性能下降

- 检查持久化存储容量与 I/O，建立数据清理与归档策略。
- 关注依赖数据库/向量库的容量与索引维护情况。

### 镜像拉取失败或启动失败

- 检查镜像仓库连通与鉴权配置；参考 [AI镜像](../llm-image) 的常见问题排查思路。
