# ComfyUI

ComfyUI 是图像生成与可视化工作流应用，支持基于节点的工作流编排，适用于图像生成、工作流复用与可视化调试。平台可将 ComfyUI 作为 AI 应用纳入统一纳管与调度体系。

## 快速开始 {#quickstart}

0. **前提**：通常需要 GPU。请先完成 [配置 NVIDIA 与 CUDA 环境](../../getting-started/setup-nvidia-cuda)，并按 [应用模板](./template) 选择合适的 GPU 显存与内存。
1. **创建应用模板**：控制台 **人工智能 → 应用 → 应用模板**，新建，类型选择 **ComfyUI**。\n   - 按需选择 [AI镜像](../llm-image) 与规格；建议同时规划持久化存储，用于保存工作流与输出结果。\n2. **创建应用实例**：控制台 **人工智能 → 应用 → 应用实例**，新建，选择刚创建的 ComfyUI 模板。\n3. **访问与验证**：进入实例详情页，在「连接信息」中获取访问地址，用浏览器打开进入 ComfyUI；导入一个示例工作流并执行一次生成，结合实例日志确认无模型加载/显存不足等错误。

## 适用场景

- 需要以可视化方式搭建与复用图像生成工作流。
- 需要对生成流程进行调试、参数化与团队协作共享。

## 依赖与资源建议

- **GPU**：通常需要 GPU 以获得可用性能。部署前建议先完成 [配置 NVIDIA 与 CUDA 环境](../../getting-started/setup-nvidia-cuda)。
- **规格选择**：参考 [应用模板](./template)，重点关注 GPU 显存与内存，避免在加载模型与生成过程中出现 OOM。
- **存储**：建议使用持久化存储保存工作流、模型（如需要）、素材与生成结果，避免实例重启后丢失数据。
- **网络**：如需拉取镜像或模型，请确保节点具备访问对应仓库的网络连通性。

## 模型与数据

- 图像生成通常需要较大的模型文件与缓存空间；建议结合 [AI模型库](../llm-inference/model-library) 统一管理模型来源与版本（如平台提供对应能力）。
- 对生成结果与工作流建议建立目录规划与清理策略，避免长期积累导致磁盘占满。

## 运维要点

- **扩缩与容量**：关注 GPU 显存、生成队列与并发；容量不足时优先扩容 GPU 或横向扩展实例。
- **升级与回滚**：通过 [AI镜像](../llm-image) 的版本管理进行升级与回滚，建议在变更前保存工作流与关键配置。
- **观测**：关注 GPU/CPU 利用率、显存占用、任务耗时与错误率；结合日志定位模型加载失败、显存不足等问题。

## 常见问题

### 1. 生成过程报错或性能很差

- 检查是否已配置 GPU 与驱动/CUDA 环境：参考 [配置 NVIDIA 与 CUDA 环境](../../getting-started/setup-nvidia-cuda)。
- 检查规格是否满足模型加载与生成所需的显存与内存峰值。

### 2. 工作流/生成结果丢失

- 检查是否已将工作流、素材与输出目录配置到持久化存储；避免仅存储在临时目录中。

### 3. 镜像/模型拉取失败

- 检查网络连通与仓库访问权限；在内网环境可考虑提前准备镜像与模型缓存（概念详见 [AI镜像](../llm-image) 与 [AI模型库](../llm-inference/model-library)）。
