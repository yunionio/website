# AI模板

定义应用的 CPU/内存/GPU/存储等资源配置，供调度与分配。

## 核心概念

控制台以 **推理模板** 或 **应用模板** 维护 AI 模板；创建实例时选择模板，即获得对应的资源配置。模板定义资源需求（CPU/内存/GPU/存储等），调度决定实例运行在哪个节点；当资源不足或约束不满足时，实例会调度失败。

常用的资源配置维度包括：

- **CPU 核数**：影响请求处理与并发能力。
- **内存**：影响模型加载与缓存；不足易 OOM 或重启。
- **GPU 数量 / 显存**：决定推理与生成性能以及可加载的模型上限（推理类 Ollama/vLLM、图像类 ComfyUI 更依赖 GPU）。
- **存储**：用于模型缓存、日志、工作流与输出结果的持久化。

## 常见问题

### 工作负载无法调度/一直等待

资源不足或约束不满足（如无符合的 GPU 节点）。确认节点资源与 GPU 环境后，再评估扩容或调整规格。

### 规格足够但仍 OOM

推理/生成存在峰值显存；按峰值规划并预留冗余。
